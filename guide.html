03

Database Container Setup

üóÑÔ∏è

MySQL Container

Authentication and user management

Pull MySQL Image
docker pull mysql:8.0
Run Container
docker run -d \
-p 3307:3306 \
--name guvi-mysql \
-e MYSQL_ROOT_PASSWORD=root \
-e MYSQL_DATABASE=guvi_auth \
mysql:8.0
Port: 3307 (avoiding conflicts with local MySQL) Database: guvi_auth
‚ÑπÔ∏è Note: Port 3307 is used because the default 3306 is often occupied by a local MySQL installation.
üçÉ

MongoDB Container

Profile and document storage

Pull & Run MongoDB
docker pull mongo docker run -d -p 27017:27017 --name guvi-mongo mongo
Port: 27017 Database: guvi_base
‚ö°

Redis Container

Session and token caching

Pull & Run Redis
docker pull redis docker run -d -p 6379:6379 --name guvi-redis redis
Port: 6379 Purpose: Session storage

Verify All Containers

docker ps

You should see: guvi-mysql, guvi-mongo, and guvi-redis running

04

Database Schema Creation

MySQL Table Setup

1

Enter MySQL Container

docker exec -it guvi-mysql mysql -u root -proot
2

Create Users Table

USE guvi_auth; CREATE TABLE users (
id INT AUTO_INCREMENT PRIMARY KEY,
name VARCHAR(100),
email VARCHAR(100) UNIQUE,
password VARCHAR(255)
);
Verify Table Creation
SHOW TABLES; SELECT * FROM users;

MongoDB Collection Setup

1

Enter MongoDB Shell

docker exec -it guvi-mongo mongosh
2

Create Database & Collection

use guvi_base db.guvi_users.insertOne({ test: "ok" })
üçÉ Auto-Creation: MongoDB automatically creates the database guvi_base and collection guvi_users upon the first insert.
